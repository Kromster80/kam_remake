 Beschreibung

Die KaM Remake Mod ist ein Versuch das originale "Knights and Merchants: The Shattered Kingdom" wiederzubeleben. Unsere Ziele sind es, das Spiel auf heutigen Computern und Betriebssystemen zum Laufen zu bringen, einen vollwertigen Multiplayermodus bereitzustellen und ebenso die vielen Fehler im Spiel zu beheben. Um dies zu machen haben wir angefangen unseren eigenen Code für die Engine zu schreiben, greifen jedoch auf viele originale Dateien zurück. Wir setzen voraus, dass ihr das original KaM Spiel besitzt (das Setup überprüft dies beim Installieren), andererseits müsst ihr es vor der Installation kaufen. Ihr bekommt es u. a. bei amazon.de, ebay.de und im Zuxxez Shop.
Das Spiel ist Freeware und von Liebhabern erstellt. Programme, die dazu benutzt werden: Delphi 6, Delphi 7, FastMM4, Lazarus, LNet, OpenGL, OpenAL, Overbyte ICS, PNGImage, zLib, MadExcept, libZPlay.
 

 Systemanforderungen

KaM Remake benötigt:
    - Microsoft Windows 98, XP, Vista oder Sieben;
    - 500MHz oder einen schnelleren Prozessor;
    - 256MB oder mehr RAM;
    - 3D Grafikkarte OpenGL 1.4 fähig;
    - Soundkarte.
 

 Installation

1. Führe den Installer aus und befolge die Anweisungen.
2. Das Paket beinhaltet die OpenAL Audiotreiber (oalinst.exe), welche ihr installieren solltet um das KaM Remake zu spielen. Die neuste Version kann unter der offiziellen OpenAL-Seite gefunden werden: http://connect.creativelabs.com/openal/default.aspx
3. Das Remake benutzt die OpenGL Rendering Engine, ihr müsst also die OpenGL-Treiber installiert haben. Besucht die Seite des Herstellers eurer Grafikkarte um die neusten Treiber zu bekommen.
4. Startet die Modifikation - KaM_Remake.exe vom Desktop oder vom Startmenü aus.
 

 Gameplay

Folgende Tastenkürzel sind verfügbar:
Esc schließt eine offene Nachricht oder Auswahl
F1-F4 öffnet die jeweilige Auswahl im Spielmenü
F5-F8 erhöht die Spielgeschwindigkeit um x10
F11 zeigt Menü und Debugging Panel an
1-9 wählt Einheiten oder Häuser aus, die den Tastenkombinationen Strg + 1-9 zugewiesen wurden
B platziert eine Markierung auf der Karte, die von Verbündeten gesehen werden kann
P pausiert das Spiel
T while pressed will show teamnames over units in multiplayer
Enf löscht die angezeigte Nachricht
Leertaste bewegt das Sichtfeld zum Ort der letzten Nachricht
←↑→↓ Pfeiltasten bewegen das Sichtfeld
Drehen des Mausrads zoomt heran und heraus
← Backspace setzt den Zoom auf 100% zurück

Zusätzliche Karten installieren:
1. Erstellt einen Unterordner im Verzeichnis 'Maps' und bennent ihn zum Beispiel "Meine neue Karte".
2. Fügt die .dat- und .map-Datei ein und benennt sie entsprechened "Meine neue Karte.dat" und "Meine neue Karte.map". Die Dateinamen sollten exakt dem Ordnernamen gleichen.
3. Ihr könnt optional auch eine Datei namens "Meine neue Karte.txt" anlegen, welche die Missionsbeschreibung und andere Informationen enthalten wird. Benutzt das selbe Format wie in den Missionen die das Remake anbietet. Keine offizielle Formatierungsinformation wird veröffentlicht, da sich das Format später wahrscheinlich noch ändern wird.

Benutzerdefinierte Musik hinzufügen:
In das Verzeichnis 'Music' könnt ihr MP3/OGG-Dateien einfügen, sie werden automatisch in die Playlist geladen.
 

 F.A.Q.

Abstürze ohne Benachrichtigung:
Wenn das KaM Remake ohne Meldung während des Startens abstürzt, sendet uns bitte die Log-Datei (contact@kamremake.com), sowie alle anderen Informationen, die ihr finden könnt und uns nützlich sein könnten.

Niedrige Framerate:
Falls das Programm eine niedrige Framerate hat, könnte es sein, dass die OpenGL-Treiber nicht mehr aktuell sind. Ihr könnt die OpenGL-Version in der Ecke links oben im Hauptmenü sehen. Es sollte mindestens Version 1.4.x sein.

Fehler im Spiel:
Sometimes when something unexpected happens you will get an "An error has occurred in the application" message. Please click the Send Bug Report button to send the crash report to us. Enter your name and email address in the form if you would like us to be able to contact you about the crash. Having the crash report will help us to fix the bug.

 Dedizierter Server
Ausführbare Serverdateien für den Multiplayermodus sind sowohl für Windows, als auch für Linux (beide x86 und 64) vorhanden. Die dedizierten Server schauen nach Verbindungen auf dem TCP-Port (Standard 56789) und arbeiten wie ein Hub, um Daten zwischen den Clients hin und her zu schicken. Der Server versteht nichts vom eigentlichen Gameplay des Spiels, er verschickt nur Datenpakete. Der erste Client/Spieler, der einen Raum betritt erhält automatisch Rechte ein Onlinespiel zu konfigurieren und zu starten ("Chef"). Wenn dieser Client seine Verbindung verliert bzw. aussteigt, erhält der Nächste in der Liste diese Rechte.
Der dedizierte Server kann in der Datei KaM_Remake_Settings.ini eingerichtet werden. Einstellungen für den Server befinden sich im Abschnitt [Server].
Um das Ganze im Netzwerk zu spielen, müsst ihr euch zuerst vergewissern, dass der Server durch das Setzen der Variable "AnnounceDedicatedServer=0" in der INI-Datei nicht parallel zum Master-Server erscheint. Startet dann den dedizierten Server und versichert euch, dass er korrekt funktioniert (das Fenster zeigt diese Meldung an "Listening on port 56789"). Beachtet, dass die Firewall den Port 56789 nicht blockieren darf. Dann könnt ihr den Server vom selben und/oder anderen Rechner/n starten (sofern der selbe Computer verwendet wird benutzt die Adresse 127.0.0.1 oder localhost).
Für den Gebrauch über das Internet müsst ihr den Port im Router freigeben (für weitere Informationen über Routerfreigaben, besucht http://portforward.com/). Sagt euren Freunden, sie sollen sich zu eurer externen IP-Adresse verbinden, diese wird auf der Homepage angezeigt http://portforward.com/. Wenn ihr auf dem selben Computer spielt, auf dem der dedizierte Server oder im selben Netzwerk, verbindet euch zur IP-Adresse wie als wenn ihr es bei einem LAN-Spiel machen würdet.

Hinweis: Der dedizierte Server wird nicht benötigt um im Multiplayermodus zu spielen. Im MP Menü auf "Starte lokalen Server" oder "Starte Internet Server" zu klicken, hat den selben Effekt. Außer du musst an einem Spiel teilnehmen, indem sich der Server nicht selbst verwalten kann. Der dedizierte Server wird nur benötigt, wenn du einen Server bereitstellen willst, ohne zu spielen. Der wichtigste Grund für den Einsatz eines dedizierten Servers ist das Anbieten eines 24/7 Hosts, bei dem sich Spieler jederzeit anmelden können. Wenn ihr in der Lage seid so einen Host anzubieten, würden wir uns sehr freuen.

Die Linux Builds wurden auf Fedora und Ubuntu getestet. Sämtliche Rückfragen betreffend der Linux Build bitte an Lewin senden (lewinjh@gmail.com).
